## 📋 问题描述

### 发现的漏洞

**类型**: Service Role Key 暴露
**位置**: `nuxt.config.ts` 第107行（修复前）
**影响**: 🔴 极高

```typescript
// ❌ 修复前（危险）
public: {
  supabaseServiceKey: process.env.SUPABASE_SERVICE_KEY
}
```

### 风险说明

Service Role Key 被暴露到浏览器前端，导致：

1. **数据泄露风险** 🔴
   - 任何访问后台管理页面的人都能获取 Service Key
   - 攻击者可以读取所有数据库数据

2. **数据篡改风险** 🔴
   - 可以修改任何数据（用户、订单、商品）
   - 可以删除数据

3. **权限绕过** 🔴
   - Service Key 拥有最高权限
   - 绕过所有 RLS（行级安全）策略

### 攻击场景

```
攻击者操作：
1. 访问 https://www.fantula.com/admin
2. 打开浏览器开发者工具（F12）
3. 查看 JavaScript 文件或 Network 请求
4. 找到并复制 Service Role Key
5. 使用该 Key 直接操作数据库

⏱️ 耗时：< 1分钟
🔓 无需密码或特殊权限
```

---

## ✅ 修复方案

### 实施方案：环境隔离

**核心思路**：
- 生产环境：完全不暴露 Service Key ✅
- 开发环境：允许暴露（受控环境，仅内部团队）

### 代码修改

```typescript
// ✅ 修复后（安全）
public: {
  supabaseServiceKey: process.env.NODE_ENV === 'production'
    ? undefined                        // 生产环境：不暴露
    : process.env.SUPABASE_SERVICE_KEY // 开发环境：可用
}
```
