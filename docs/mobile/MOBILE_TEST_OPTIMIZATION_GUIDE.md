# 移动端测试与优化规范

> **版本**: V1.0 | **更新时间**: 2026-02-04
> **适用对象**: AI 测试工程师

---

## 🔴 核心红线 (绝对禁止)

### ⚠️ 最重要的规则

```
移动端 = PC端逻辑 + 移动端UI布局

逻辑不能改，只能改布局！
```

### 禁止项

| 禁止 | 说明 |
|------|------|
| 新增 API | 必须使用现有 PC 端接口 |
| 复制业务逻辑 | 必须 import PC 端逻辑 |
| 创建新状态 | 必须使用共享 Store |
| 改变操作流程 | 与 PC 端保持一致 |
| 使用 Element Plus | 移动端禁止使用 |

## ✅ 允许的优化范围

### 1. 布局优化

| 允许 | 示例 |
|------|------|
| 单列布局 | 移动端屏幕适配 |
| 底部导航 | 移动端专属交互 |
| 抽屉菜单 | 侧滑菜单 |
| 堆叠布局 | 卡片堆叠 |

### 2. 交互优化

| 允许 | 示例 |
|------|------|
| 触摸手势 | 滑动、长按 |
| 下拉刷新 | 移动端体验 |
| 上拉加载 | 无限滚动 |
| 底部弹窗 | Action Sheet |

### 3. 性能优化

| 允许 | 示例 |
|------|------|
| 图片优化 | 响应式图片 |
| 懒加载 | 按需加载 |
| 骨架屏 | 加载体验 |
| 缓存 | 减少请求 |

## 📊 页面测试流程

### 扫描检查清单

```markdown
## 页面: [页面名称]
路径: `pages/mobile/xxx.vue`

### 1. 与 PC 端一致性检查 (最重要!)
- [ ] 使用相同 API (api/client/)
- [ ] 使用相同 Store
- [ ] 使用相同 Composable
- [ ] 业务逻辑一致
- [ ] 数据展示一致

### 2. 功能验证
- [ ] 页面正常加载
- [ ] 数据正确显示
- [ ] 交互正常响应

### 3. 移动端体验
- [ ] 布局适配正确
- [ ] 触摸交互正常
- [ ] 滚动流畅
- [ ] 加载体验良好

### 4. 代码质量
- [ ] 无复制的业务逻辑
- [ ] 无新增的 API 调用
- [ ] 无 Element Plus 组件

### 5. 优化建议
[列出发现的可优化项]
```

## 🔗 PC/Mobile 一致性验证

### 测试方法

```markdown
## 一致性测试: [页面名称]

### 数据源验证
对比 PC 和 Mobile 相同订单/商品:
- [ ] 数据内容完全一致
- [ ] 状态显示完全一致
- [ ] 按钮状态完全一致

### 操作验证
- [ ] 相同操作产生相同结果
- [ ] 状态变更后两端同步
```

## 🧩 移动端专属组件

### 可用组件 (components/mobile/)

| 组件 | 用途 |
|------|------|
| `MobileRefundingCard` | 退款处理中卡片 |
| `MobileCancelRefundSheet` | 取消退款底部弹窗 |
| 底部导航 | 移动端主导航 |
| ActionSheet | 底部操作菜单 |

### 禁止使用

| 组件 | 原因 |
|------|------|
| Element Plus 任何组件 | 太重，移动端禁止 |
| PC 端布局组件 | 不适合移动端 |
