> **版本**: V1.0 | **更新时间**: 2026-02-08
> **目的**: AI 助手执行移动端组件标准化的详细工作手册

---

## 🎯 概述

本文档定义了将移动端非标准组件迁移到全局基础组件的**完整工作流程**。
AI 助手在执行标准化任务时，必须严格按照以下步骤操作。

## 🔄 二、标准化工作流程

### Phase 1: 任务接收与分析

```
┌─────────────────────────────────────────────────────────────┐
│ 步骤 1.1: 确认任务范围                                        │
├─────────────────────────────────────────────────────────────┤
│ - 确认要标准化的组件名称                                      │
│ - 确认标准化类型 (弹窗/抽屉/按钮)                              │
│ - 确认目标基础组件 (MobileBaseModal/MobileBaseSheet)          │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 步骤 1.2: 扫描当前组件                                        │
├─────────────────────────────────────────────────────────────┤
│ 命令: view_file 打开目标组件                                  │
│                                                             │
│ 记录以下信息:                                                │
│ - Props 列表 (visible, title, 等)                            │
│ - Emits 列表 (close, confirm, 等)                            │
│ - 模板结构 (header, body, footer)                            │
│ - 业务逻辑 (API 调用, 状态管理)                               │
│ - 样式定义 (<style> 部分)                                    │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 步骤 1.3: 检查基础组件                                        │
├─────────────────────────────────────────────────────────────┤
│ 命令: view_file 打开基础组件                                  │
│                                                             │
│ 确认:                                                        │
│ - 基础组件是否存在                                           │
│ - 基础组件的 Props 是否满足需求                               │
│ - 是否需要扩展基础组件                                       │
└─────────────────────────────────────────────────────────────┘
```

---

### Phase 2: 制定迁移计划

```
┌─────────────────────────────────────────────────────────────┐
│ 步骤 2.1: 生成迁移报告                                        │
├─────────────────────────────────────────────────────────────┤
│ 格式:                                                        │
│                                                             │
│ ## 迁移计划: [组件名]                                         │
│                                                             │
│ ### 当前状态                                                 │
│ - 模板行数: XX 行                                            │
│ - 样式行数: XX 行                                            │
│ - 使用自定义: [列表]                                         │
│                                                             │
│ ### 迁移内容                                                 │
│ 1. 替换模板: 使用 MobileBaseModal                            │
│ 2. 删除样式: .modal-overlay, .modal-content 等               │
│ 3. 保留逻辑: [业务逻辑说明]                                   │
│                                                             │
│ ### 风险评估                                                 │
│ - 低风险: 仅样式变更                                         │
│ - 中风险: 涉及插槽调整                                       │
│ - 高风险: 涉及业务逻辑                                       │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 步骤 2.2: 用户确认                                           │
├─────────────────────────────────────────────────────────────┤
│ ⚠️ 必须等待用户确认后才能继续                                 │
│                                                             │
│ 提问模板:                                                    │
│ "以上迁移计划是否可以执行？                                   │
│  - 是: 我将开始执行迁移                                       │
│  - 否: 请告诉我需要调整的地方"                                │
└─────────────────────────────────────────────────────────────┘
```

---

### Phase 3: 执行迁移

```
┌─────────────────────────────────────────────────────────────┐
│ 步骤 3.1: 备份原组件 (可选)                                   │
├─────────────────────────────────────────────────────────────┤
│ 如果组件复杂，建议先复制一份备份                               │
│                                                             │
│ 命令: cp 组件.vue 组件.vue.bak                               │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 步骤 3.2: 修改模板                                           │
├─────────────────────────────────────────────────────────────┤
│ 修改内容:                                                    │
│                                                             │
│ 1. 替换外层结构:                                             │
│    - 移除 <Teleport>                                        │
│    - 移除 <Transition>                                      │
│    - 移除 .modal-overlay                                    │
│    - 移除 .modal-content                                    │
│                                                             │
│ 2. 使用基础组件:                                             │
│    <MobileBaseModal                                         │
│      v-model:visible="visible"                              │
│      title="标题"                                           │
│      :type="type"                                           │
│      @confirm="handleConfirm"                               │
│      @close="handleClose"                                   │
│    >                                                        │
│      <!-- 内容插槽 -->                                       │
│    </MobileBaseModal>                                       │
│                                                             │
│ 3. 保留业务逻辑:                                             │
│    - 表单字段                                               │
│    - 校验逻辑                                               │
│    - API 调用                                               │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 步骤 3.3: 清理样式                                           │
├─────────────────────────────────────────────────────────────┤
│ 删除以下自定义样式:                                          │
│                                                             │
│ - .modal-overlay { ... }                                    │
│ - .modal-content { ... }                                    │
│ - .modal-header { ... }                                     │
│ - .modal-footer { ... }                                     │
│ - .close-btn { ... }                                        │
│ - .cancel-btn { ... }                                       │
│ - .save-btn { ... }                                         │
│ - 相关动画 @keyframes                                        │
│                                                             │
│ 保留以下样式:                                                │
│ - 业务相关的表单样式                                         │
│ - 特殊警告框样式                                             │
│ - 自定义内容布局                                             │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 步骤 3.4: 添加导入                                           │
├─────────────────────────────────────────────────────────────┤
│ 确保 <script setup> 中有:                                    │
│                                                             │
│ import MobileBaseModal from                                 │
│   '@/components/mobile/base/MobileBaseModal.vue'            │
└─────────────────────────────────────────────────────────────┘
```

## 📝 三、标准化示例

### 3.1 弹窗标准化示例

**原始代码** (DeleteAccountModal.vue):
```vue
<template>
  <Teleport to="body">
    <div v-if="visible" class="modal-overlay" @click.self="handleClose">
      <div class="modal-content">
        <div class="modal-header">
          <h3 class="modal-title">注销账号</h3>
          <button class="close-btn" @click="handleClose">×</button>
        </div>
        <div class="modal-body">
          <!-- 内容 -->
        </div>
        <div class="modal-footer">
          <button class="cancel-btn" @click="handleClose">取消</button>
          <button class="save-btn" @click="handleConfirm">确认</button>
        </div>
      </div>
    </div>
  </Teleport>
</template>

<style scoped>
.modal-overlay { ... }
.modal-content { ... }
/* 大量样式代码 */
</style>
```

**标准化后**:
```vue
<template>
  <MobileBaseModal
    v-model:visible="visible"
    title="注销账号"
    type="danger"
    confirm-text="确认注销"
    :loading="loading"
    @confirm="handleConfirm"
    @close="handleClose"
  >
    <!-- 仅保留业务内容 -->
    <div class="warning-box">
      <p>警告：此操作不可恢复</p>
    </div>
    <div class="form-item">
      <input v-model="code" placeholder="验证码" />
    </div>
  </MobileBaseModal>
</template>

<style scoped>
/* 仅保留业务相关样式 */
.warning-box { ... }
.form-item { ... }
</style>
```

## ⚠️ 四、注意事项

### 4.1 禁止事项

| 禁止 | 原因 |
|------|------|
| ❌ 不经确认直接执行 | 可能导致功能破坏 |
| ❌ 删除业务逻辑 | 标准化仅涉及样式 |
| ❌ 修改 Props 接口 | 可能影响父组件 |
| ❌ 跳过验证步骤 | 必须确保功能正常 |

### 4.2 必须事项

| 必须 | 原因 |
|------|------|
| ✅ 生成迁移计划 | 便于审核和回滚 |
| ✅ 等待用户确认 | 用户有最终决定权 |
| ✅ 保留业务逻辑 | 标准化不改功能 |
| ✅ 生成验证清单 | 确保迁移质量 |
