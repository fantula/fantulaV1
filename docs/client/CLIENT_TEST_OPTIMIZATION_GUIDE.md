# PC 客户端测试与优化规范

> **版本**: V1.0 | **更新时间**: 2026-02-04
> **适用对象**: AI 测试工程师

---

## 🔴 核心红线 (绝对禁止)

### 禁止项

| 禁止 | 说明 |
|------|------|
| 改变页面操作流程 | 按钮效果、跳转逻辑不可变 |
| 新增/修改 API | 必须使用现有接口 |
| 复制业务逻辑 | PC/Mobile 共享同一逻辑 |
| 修改数据获取方式 | 保持现有 Store/Composable |
| 改变订单详情核心逻辑 | ⚠️ 特别注意 |

---

## ✅ 允许的优化范围

### 1. UI/UX 优化

| 允许 | 示例 |
|------|------|
| 布局优化 | 调整间距、对齐 |
| 动画优化 | 过渡动画、加载效果 |
| 响应式优化 | 不同屏幕适配 |
| 骨架屏 | 加载状态体验 |

### 2. 性能优化

| 允许 | 示例 |
|------|------|
| 懒加载 | 图片、组件懒加载 |
| 请求优化 | 合并请求、缓存 |
| 渲染优化 | 减少重渲染 |
| 代码分割 | 按需加载 |

### 3. 代码精简

| 允许 | 示例 |
|------|------|
| 删除死代码 | 未使用变量、函数 |
| 提取公共逻辑 | 重复代码抽成函数 |
| 优化类型 | 添加 TypeScript 类型 |
| 格式统一 | 使用 useBizFormat |

## ⭐ 重点页面: 订单详情

> **路径**: `pages/profile/order/[id].vue`
> **详细文档**: `ORDER_DETAIL.md`

### 为什么特别重要

1. **逻辑最复杂**: 状态机、多类型切换、操作按钮状态
2. **组件最多**: 6+ 个子组件
3. **PC/Mobile 共享**: 逻辑必须一致

### 测试要点

```markdown
## 订单详情测试

### 状态覆盖测试
- [ ] 待发货 (pending_delivery)
- [ ] 使用中 (active)
- [ ] 已过期 (expired)
- [ ] 已取消 (cancelled)
- [ ] 退款中 (refunding)

### 类型覆盖测试
- [ ] 虚拟充值 (virtual)
- [ ] 账号合租 (shared_account)
- [ ] 一次性CDK (one_time_cdk)

### 操作按钮测试
- [ ] 续费按钮显隐
- [ ] 退款按钮状态
- [ ] 取消退款流程

### 共享逻辑验证
- [ ] 与移动端数据一致
- [ ] 与移动端状态逻辑一致
```
