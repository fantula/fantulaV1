# 后台核心操作流程 (Admin Core Workflow)

> **定位**: 功能操作手册，指导运营人员如何使用后台核心模块。
> **适用**: 运营人员、测试人员、产品经理。

---

## 一、 订单管理 (Order Management)

### 1.1 订单履约
所有订单默认自动处理。
*   **虚拟充值**: 支付后自动调用充值 API。
    *   *异常处理*: 如自动充值失败 (状态 `failed`)，需人工在详情页点击【人工补单】，手动确认服务已完成后将状态改为 `completed`。
*   **账号合租**: 支付后自动分配车位 (`slot`)。
*   **激活码**: 支付后自动发放 `code`。

### 1.2 退款处理 (`orders/refund`)
1.  用户发起的退款申请会出现在【订单管理 -> 退款管理】。
2.  **审核通过**:
    *   点击【同意退款】。
    *   系统自动退款至用户原支付渠道（或余额）。
    *   **关键**: 关联的资源（CDK库存/车位）会自动释放，回归可售状态。
3.  **审核拒绝**: 填写理由，订单恢复正常。

## 二、 商品与资源管理 (Product & Assets)

此模块逻辑严格遵循：`分类 -> 商品 -> SKU -> CDK` 层级。

### 2.1 发布新商品
1.  **分类**: 在【商品管理 -> 分类管理】确认分类存在。
2.  **商品**: 在【商品管理 -> 发布商品】创建商品主体，设置名称、图片、介绍。
3.  **规格 (SKU)**: 在商品编辑页，添加 SKU（如 "30天 4K版"）。
    *   **重要**: 必须指定 SKU 类型 (`虚拟/合租/卡密`)。

### 2.2 补充库存 (CDK)
库存是独立的实体，必须关联 SKU。
1.  进入【CDK 管理 -> 发布资源】。
2.  选择对应的【商品】和【SKU】。
3.  录入数据：
    *   **卡密**: 批量粘贴文本。
    *   **虚拟**: 填写库存数字。
    *   **合租**: 填写账号/密码及【最大车位数】(Max Slots)。

## 三、 用户与权限 (User & Permission)

### 3.1 创建管理员
1.  进入【用户管理 -> 账号列表】 -> 【新增用户】。
2.  填写邮箱、密码、名称。
3.  **绑定部门**: 选择所属部门（决定其权限）。
    *   新用户默认可用邮箱+密码登录。

### 3.2 权限变更
1.  进入【用户管理 -> 部门管理】。
2.  修改对应部门的权限勾选。
3.  保存后，该部门下所有成员的权限 **即时生效**（下次刷新页面或重新登录后）。

---

## 四、 营销与工单 (Marketing & Support)

### 4.1 优惠券配置
*   **立减券**: 满 100 减 10。
*   **额度券**: 充值送余额（充值卡）。
    *   *注意*: 额度券领取后直接增加用户钱包余额，无过期概念。

### 4.2 工单处理
1.  在【工单管理】查看用户咨询。
2.  右侧聊天框回复。
3.  问题解决后，点击【标记已解决】，工单关闭。
