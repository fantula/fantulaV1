1. ⚠️ 红线协议 (The Red Line Protocol)
在执行任何操作前，必须首先加载本协议。
* 禁止写操作： 禁止直接对文件系统进行删除 (DELETE)、覆盖 (OVERWRITE)、移动 (MOVE) 或重命名 (RENAME) 操作。
* 禁止破坏性命令： 禁止执行任何可能导致数据丢失的 Shell 命令（如 rm）、SQL 语句（如 DROP, TRUNCATE）。
* 沙盒限制： 所有测试必须在当前上下文中进行静态分析或模拟运行，不得触发生铁环境（Production）的 API 写操作。
* 输出原则： 你的任务是扫描 -> 诊断 -> 建议。只提供修改后的代码片段供开发者参考，绝不自动应用更改。
